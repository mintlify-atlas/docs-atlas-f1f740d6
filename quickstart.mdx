---

<Info>
  PvZ Portable does not include game assets. You need `main.pak` and the `properties/` folder from a legally purchased copy of Plants vs. Zombies: GOTY Edition before you can play. See the [game assets guide](/game-assets) for details.
</Info>

<Steps>
  <Step title="Download a prebuilt binary">
    Go to the [GitHub Releases page](https://github.com/wszqkzqk/PvZ-Portable/releases) and download the archive for your platform.

    Prebuilt binaries are available for Linux, Windows, macOS, Android (APK), iOS (unsigned IPA), WebAssembly, and Nintendo Switch.
  </Step>
  <Step title="Obtain your game assets">
    You need `main.pak` and the `properties/` folder from your legally purchased copy of Plants vs. Zombies: GOTY Edition.

    See the [game assets guide](/game-assets) for instructions on where to buy the game and how to locate these files after installation.
  </Step>
  <Step title="Place assets next to the executable">
    Copy `main.pak` and the `properties/` folder into the same directory as the `pvz-portable` executable.

    ```
    pvz-portable          # the binary
    main.pak              # game data archive
    properties/           # game strings and config
    ```

    The game searches for resources relative to the executable's directory, so you can launch it from any working directory and it will still find them.

    <Tip>
      You can also use extracted (unpacked) asset data instead of `main.pak` if you prefer to work with individual files directly.
    </Tip>
  </Step>
  <Step title="Run the game">
    Launch the executable directly. No installer is required.

    <Tabs>
      <Tab title="Linux / macOS">
        ```bash
        ./pvz-portable
        ```
      </Tab>
      <Tab title="Windows">
        Double-click `pvz-portable.exe`, or run it from a terminal:

        ```bat
        pvz-portable.exe
        ```
      </Tab>
    </Tabs>
  </Step>
</Steps>

## Default data paths

Per-user writable files (settings, savegames, compiled caches, screenshots) are stored in the OS-recommended application data directory under `io.github.wszqkzqk/PvZPortable`.

<Tabs>
  <Tab title="Linux">
    ```
    ~/.local/share/io.github.wszqkzqk/PvZPortable/
    ```
  </Tab>
  <Tab title="Windows">
    ```
    %APPDATA%\io.github.wszqkzqk\PvZPortable\
    ```
  </Tab>
  <Tab title="macOS">
    ```
    ~/Library/Application Support/io.github.wszqkzqk/PvZPortable/
    ```
  </Tab>
</Tabs>

This directory contains the following subfolders and files:

| Path | Description |
|---|---|
| `userdata/` | Player save files (profiles, adventure progress, coins, Zen Garden). |
| `cache64/` or `cache32/` | Compiled binary caches (reanimation / compiled definitions). These are local startup artifacts and are **not** portable across platforms. The game transparently recompiles them when a cache or schema check fails. The 64-bit build uses `cache64/`; the 32-bit build uses `cache32/`. |
| `registry.regemu` | Settings and registry emulation. |

## Command-line parameters

You can override the default resource and save data directories at launch.

| Parameter | Description |
|---|---|
| `-resdir="<path>"` | Set the resource directory where `main.pak` and `properties/` are located. Does not affect where the game saves data. |
| `-savedir="<path>"` | Set the save data directory where settings, savegames, caches, and screenshots are stored. Overrides the default OS-recommended path. |

<Warning>
  You **must** use the format `-param="<Your Path>"`. Space-separated values such as `-resdir path` are **not** supported.
</Warning>

<CodeGroup>

```bash Linux / macOS
./pvz-portable -resdir="/mnt/games/pvz-assets" -savedir="/mnt/saves/pvz"
```

```bat Windows
pvz-portable.exe -resdir="D:\Games\PvZ\assets" -savedir="D:\Games\PvZ\saves"
```

</CodeGroup>
